Task 1: Decrypt String

This week's first task presents an interesting decoding challenge. We're given a string of digits and # characters, where 'a'-'i' are mapped to '1'-'9', and 'j'-'z' are mapped to '10#'-'26#'.

The Strategy: The key difficulty lies in the overlap. A '1' followed by a '0' could be 'aa' (1, 0) if 0 were valid, or it could be the start of '10#'. However, since 'j'-'z' always end with a #, we can use a non-greedy regex that prioritizes the three-character pattern.

By using the regex (\d{2})#|(\d), we first attempt to match the two-digit-plus-hash pattern. If that fails, we fall back to a single digit. This ensures that '10#' is correctly identified as 'j' rather than '1' and '0'.

Perl Implementation
sub decrypt_string {
    my ($str) = @_;
    # Global replacement with an evaluation block
    $str =~ s/(\d{2})#|(\d)/
        if (defined $1) {
            chr(ord('a') + $1 - 1)
        } else {
            chr(ord('a') + $2 - 1)
        }
    /ge;
    return $str;
}
Python Implementation
def decrypt_string(s: str) -> str:
    # Python's re.sub accepts a lambda for complex logic
    return re.sub(r'(\d{2})#|(\d)',
                  lambda m: chr(ord('a') + int(m.group(1)) - 1) if m.group(1)
                            else chr(ord('a') + int(m.group(2)) - 1),
                  s)

Task 2: Goal Parser

The second task is a classic "parser" problem, though it's simple enough to solve with direct string replacement. We need to interpret G as "G", () as "o", and (al) as "al".

In both languages, the order of replacement is important if we're not using a formal lexer. Replacing (al) before () is generally safer, although in this specific case, the patterns are distinct enough that it doesn't cause conflicts.

Perl Implementation
sub goal_parser {
    my ($str) = @_;
    # Simple global substitutions
    $str =~ s/\(al\)/al/g;
    $str =~ s/\(\)/o/g;
    return $str;
}
Python Implementation
def goal_parser(s: str) -> str:
    # Chained string replacements are efficient for this task
    return s.replace("(al)", "al").replace("()", "o")

This approach is O(n) and highly readable, making it the idiomatic choice for such a straightforward mapping.
